💾 Backup & Restore
AES-256 encrypted local backups — Phase 08
📤 Export Encrypted Backup
🔒
Create Encrypted Local Backup
Exports the full SQLite database — all transactions, clients, audit logs, settings — encrypted with AES-256-GCM using a password you choose. The password is never stored anywhere.
File format: The .demsbackup file contains a DEMS magic header, PBKDF2-derived key material (100,000 iterations, random 16-byte salt), a random 12-byte AES-GCM IV, and the ciphertext. The password is never written to disk or localStorage — lose it and the backup cannot be recovered.
📥 Restore from Backup
⚠️ Critical warning: Restoring a backup completely overwrites your current database — all transactions, clients, and audit logs will be replaced. This action cannot be undone. After restore the system logs you out and reloads.

🔒 Encryption Specification
🔑Key derivation: PBKDF2-SHA-256 with 100,000 iterations and a random 16-byte salt per backup
🔐Cipher: AES-256-GCM — authenticated encryption, tamper detection built-in
🎲IV: random 12 bytes per backup — never reused
🪪File header: DEMS magic + version byte for integrity check before decryption
🚫Password is never stored, hashed, or transmitted — it exists only in memory during the operation
☁️ Cloud Backup — Coming in Phase 08.2
Firebase-based automatic daily encrypted cloud backup is planned as the next increment of Phase 08. It will use the same AES-256-GCM scheme with a user-managed cloud passphrase stored in no server.
